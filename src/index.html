git什么

要想使用一个工具，显然要了解它是什么？

git是什么?
git是源代码管理工具。简单地来说就是帮你管理你的代码。
简单地来说就是帮你管理你的代码。就像定期借钱一样，它会把借多少钱，借钱缘由，是否归还等等相关事宜都记录清楚。当然，你也可以拿回一部分。

为什么要学习git

对于一个项目来说，版本的更迭与管理会是一件繁琐的事情，而git能够提供一个很好的仓库给你！这还只是其中一点。

还有重要的一点就是团队协作开发，使用git工具，能够让不同地区的人，不同时区的人有机会一起搭建项目，一起完成项目的开发，这也是它为什么那么受欢迎的地方之一了。

同样，企业上的项目开发管理也是用git进行管理，所以，很有必要学习一下

使用git

站在岸上学不会游泳，所以，要想熟练掌握一个工具，那必须得拿到工具。官网 https://git-scm.com/download/win

在github网上注册一个账号，把它想象成一个大仓库，而且是免费的，听上去多么诱人。我们的项目都将会使用github托管。
点击github页面上的new repository按钮，然后输入文件名。最后点击create repository
接着，你会看到一些命令行的操作，别急，跟着我的节奏走，你会弄明白的~

git命令

注意：以下git命令行都要在项目根目录进行

git init

项目初始化操作，执行过后会根目录多了.git文件，这是记录项目文件日志的仓库

还有一个变化就是命令行末端多了（master）

这一步相当于建本地仓库

git add README.md

git add 命令是将有变动的文件添加到本地仓库

这一步相当于把有变化的东西存在本地仓库。

git commit -m "comment"

提交项目日志

commit 提交¸-m的含义是message,comment是指日志内容

这一步相当于记录了存入仓库有什么物品，用来干嘛，怎么弄来的等等

git remote add origin URL

remote远程的意思，这里指的是github(小的这么理解而已)

add origin URL;设置URL为默认主机，其实就是把URL那个服务器当做你的远程项目仓库

这一步相当于在远处建立一个总仓库

git push -u origin master

push推，把本地文件上传到远程主机

这一步是把记录有变动的、仓库里的东西推到远处的总仓库

git status

status状态，显示本地文件的变更情况

git log

log,日志，记录以往的历史，用于查询

git diff

difference,区别，查看此次与上一版本的区别

git reset --hard hash值前6位

reset，重置，返回上一版本

git clone ***.git(网址，一般有复制)

clone,复制，从远程上下载项目到本地

使用github托管页面

github里有免费的域名可以挂项目，展示项目，听上去多诱人~
git brance gh-pages：建立gh-pages分支，这名字是固定的
git checkout gh-pages;切换当前分支为gh-pages
git push -u origin gh-pages;将本地的文件上传到gh-pages分支
username(你在github上的用户名).github.io/仓库名（repository name）;通过这个网址就可以访问你的项目了，怎样，到这里，是不是挺期待的~
